## Get Your First Image in 3 Steps

This guide will take you from zero to your first generated image in under a minute.

<Steps>
  <Step title="Run the Setup Wizard">
    The setup wizard handles authentication, MCP configuration, and default settings:
    
    ```bash
    pixelmuse setup
    ```
    
    This will:
    - Open your browser to authenticate (or prompt for an API key)
    - Optionally configure the MCP server for Claude Code, Cursor, or Windsurf
    - Let you customize default model and settings (or use smart defaults)
    
    <Note>
      New accounts include **15 free credits**. Most models cost 1 credit per generation.
    </Note>
  </Step>
  
  <Step title="Generate Your First Image">
    Create an image with a simple prompt:
    
    ```bash
    pixelmuse "a cat floating through space"
    ```
    
    You'll see:
    - A spinner showing generation progress
    - Credit cost and remaining balance
    - The saved file path
    - An inline preview (if you have [chafa](https://hpjansson.org/chafa/) installed)
    
    Example output:
    ```
    ✔ Generated in 3.2s · 1 credit charged (remaining: 14) · private
      Saved to a-cat-floating-through-space.png
    ```
  </Step>
  
  <Step title="Customize Your Generation">
    Try different models, aspect ratios, and styles:
    
    ```bash
    pixelmuse "neon cityscape at night" -m recraft-v4 -a 16:9
    ```
    
    Or save to a specific path:
    
    ```bash
    pixelmuse "app icon, minimal" -a 1:1 -o icon.png
    ```
  </Step>
</Steps>

## Common Use Cases

### Choose a Model

Pixelmuse supports multiple AI models. Use the `-m` flag to select one:

<CodeGroup>
```bash Nano Banana 2 (Default)
pixelmuse "sunset over mountains" -m nano-banana-2
# 1 credit · Fast · Great for text rendering
```

```bash Google Imagen 3
pixelmuse "photorealistic portrait" -m imagen-3
# 1 credit · Realistic photos · Complex compositions
```

```bash Recraft V4
pixelmuse "modern logo design" -m recraft-v4
# 1 credit · Typography · Design · Composition
```

```bash Flux Schnell
pixelmuse "quick mockup concept" -m flux-schnell
# 1 credit · Speed · Ideation
```
</CodeGroup>

See all available models:

```bash
pixelmuse models
```

Output:
```
Model                  Credits    Best For
──────────────────────────────────────────────────────────────────────
Nano Banana 2          1          Speed, text rendering, world knowledge
Nano Banana Pro        4          Text rendering, real-time info, multi-image editing
Flux Schnell           1          Quick mockups, ideation
Google Imagen 3        1          Realistic photos, complex compositions
Recraft V4             1          Typography, design, composition
Recraft V4 Pro         7          High-res design, art direction
```

### Select Aspect Ratio

Generate images in different dimensions:

<CodeGroup>
```bash Square (1:1)
pixelmuse "app icon, gradient" -a 1:1
```

```bash Landscape (16:9)
pixelmuse "hero banner for website" -a 16:9
```

```bash Portrait (9:16)
pixelmuse "phone wallpaper, abstract" -a 9:16
```

```bash Ultrawide (21:9)
pixelmuse "cinematic landscape" -a 21:9
```
</CodeGroup>

Supported aspect ratios: `1:1`, `16:9`, `9:16`, `3:2`, `2:3`, `4:5`, `5:4`, `3:4`, `4:3`, `21:9`, `9:21`

### Apply Styles

Add visual style presets to your generations:

<CodeGroup>
```bash Realistic
pixelmuse "portrait of a chef" -s realistic
```

```bash Anime
pixelmuse "mountain landscape" -s anime
```

```bash Artistic
pixelmuse "abstract composition" -s artistic
```

```bash No Style (Default)
pixelmuse "sunset" -s none
```
</CodeGroup>

### Save to Specific Path

By default, images save to your current directory with a slugified prompt name. Override with `-o`:

```bash
pixelmuse "hero image" -o ./public/hero.png
pixelmuse "avatar" -o ~/Desktop/avatar.jpg
```

<Note>
  Pixelmuse auto-detects image format from the file extension (`.png`, `.jpg`, `.jpeg`, `.webp`).
</Note>

## Developer Workflows

### Pipe from stdin

Generate images from file contents or command output:

```bash
echo "hero banner for SaaS landing page" | pixelmuse -o hero.png
cat prompt.txt | pixelmuse -m recraft-v4
```

### JSON Output for Scripting

Get machine-readable output:

```bash
pixelmuse --json "logo concept" | jq
```

Output:
```json
{
  "id": "gen_abc123xyz",
  "status": "succeeded",
  "model": "nano-banana-2",
  "prompt": "logo concept",
  "credits_charged": 1,
  "visibility": "private",
  "elapsed_seconds": 3.2,
  "output_path": "logo-concept.png"
}
```

Extract just the path:

```bash
OUTPUT=$(pixelmuse --json "icon" | jq -r .output_path)
echo "Saved to: $OUTPUT"
```

### Watch Mode for Iteration

Auto-regenerate when a prompt file changes:

```bash
pixelmuse --watch prompt.txt -o output.png
```

Now edit `prompt.txt` in your editor. Every time you save, Pixelmuse regenerates the image:

```
Watching /path/to/prompt.txt for changes... (Ctrl+C to stop)
✔ [14:23:45] Saved to output.png (3.1s)
✔ [14:24:02] Saved to output.png (2.9s)
```

<Note>
  Perfect for iterating on prompts — edit text, see the result instantly.
</Note>

### Additional Flags

<CodeGroup>
```bash Copy to Clipboard
pixelmuse "avatar" --clipboard
```

```bash Open in System Viewer
pixelmuse "landscape" --open
```

```bash Skip Terminal Preview
pixelmuse "icon" --no-preview
```

```bash Don't Save to Disk
pixelmuse "test" --no-save
```

```bash Make Public
pixelmuse "artwork" --public
```
</CodeGroup>

See your recent generations:

```bash
pixelmuse history
```

Output:
```

──────────────────────────────────────────────────────────────────────────────────────────

gen_def456uvw...   ● recraft-v4 🔒         neon cityscape at night           1 day ago
```

Open a generation in your system viewer:

```bash
pixelmuse open gen_abc123xyz
```

## Launch the Interactive TUI

For a visual interface with menus, galleries, and wizards:

```bash
pixelmuse ui
```

The TUI provides:
- Visual model selection
- Guided generation wizard
- Gallery browser with previews
- Account overview
- Prompt template management

<Note>
  Use arrow keys to navigate, Enter to select, and Esc to go back.
</Note>

## Use Prompt Templates

Create reusable prompts with variables:

<Steps>
  <Step title="Create a Template">
    ```bash
    pixelmuse template init blog-thumbnail
    ```
    
    This creates `~/.config/pixelmuse-cli/prompts/blog-thumbnail.yaml`
  </Step>
  
  <Step title="Edit the Template">
    Open the file and customize:
    
    ```yaml
    name: Blog Thumbnail
    description: Dark-themed blog post thumbnail
    prompt: >
      A cinematic {{subject}} on a dark gradient background,
      dramatic lighting, 8K resolution
    defaults:
      model: nano-banana-2
      aspect_ratio: "16:9"
    variables:
      subject: "code editor with syntax highlighting"
    tags: [blog, thumbnail, dark]
    ```
  </Step>
  
  <Step title="Use the Template">
    Generate with variable substitution:
    
    ```bash
    pixelmuse template use blog-thumbnail --var subject="React hooks guide"
    ```
    
    The template's default model and aspect ratio are applied automatically.
  </Step>
</Steps>

List all templates:

```bash
pixelmuse template list
```

Show template details:

```bash
pixelmuse template show blog-thumbnail
```

## Tips for Better Results

<Note>
  **Be specific** — "A red 1967 Ford Mustang on a coastal highway at sunset, cinematic lighting" works better than "car"
</Note>

<Note>
  **Choose the right model** — Use `imagen-3` for photorealism, `recraft-v4` for typography/design, `nano-banana-2` for speed
</Note>

<Note>
  **Iterate with watch mode** — Use `--watch` to refine prompts in real-time
</Note>

<Note>
  **Check model costs** — Run `pixelmuse models` to see credit costs before generating
</Note>
