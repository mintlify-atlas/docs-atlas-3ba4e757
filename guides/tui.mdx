The Pixelmuse TUI is a full-screen terminal interface for browsing models, generating images with guided wizards, viewing your gallery, and managing your account — all without memorizing commands.

## Launching the TUI

```bash
pixelmuse ui
```

This opens an interactive interface built with [Ink](https://github.com/vadimdemedes/ink) (React for the terminal).

## Navigation

The TUI uses keyboard-based navigation:

- **Arrow keys** or **j/k**: Navigate menus and lists
- **Enter**: Select an item or confirm action
- **Tab**: Move between form fields
- **Esc**: Go back or cancel
- **q**: Quit the TUI

## Features

### Home screen

The home screen shows:
- Your current credit balance
- Recent generations
- Quick actions (Generate, Models, Account, Gallery)

### Generation wizard

Select **Generate** to open a step-by-step wizard:

1. **Enter prompt**: Type your image description
2. **Choose model**: Browse available models with descriptions and costs
3. **Select aspect ratio**: Pick from all supported ratios with visual previews
4. **Apply style**: Choose realistic, anime, artistic, or none
5. **Review**: Confirm settings and credit cost
6. **Generate**: Watch progress in real-time

Once complete, the image opens in your system viewer (if enabled).

### Models browser

Select **Models** to view:
- All available models
- Credit costs per generation
- Model strengths (text rendering, realism, speed, etc.)
- Supported aspect ratios

### Gallery viewer

Select **Gallery** to:
- Browse your recent generations
- Filter by status (succeeded, failed, pending)
- View generation metadata (model, prompt, credits charged)
- Open images in your system viewer
- Copy image paths to clipboard

### Account screen

Select **Account** to view:
- Your email and plan
- Total credits (subscription + purchased)
- Credit breakdown
- Rate limits
- Usage statistics (if available)

### Prompt templates

Select **Templates** to:
- List all saved templates
- Create new templates
- Edit existing templates
- Use templates with variable substitution

## When to use the TUI vs. CLI

| Use TUI when... | Use CLI when... |
|-----------------|------------------|
| Exploring models and features | You know exactly what you want |
| Learning aspect ratios and styles | Scripting or automation |
| Browsing your gallery | Piping from other tools |
| Managing templates visually | Running in CI/CD |
| You prefer guided workflows | You prefer flags and speed |

## Visual features

The TUI includes:

- **Color-coded status**: Green for success, red for errors, yellow for warnings
- **Real-time progress**: Elapsed time during generation
- **Credit previews**: See cost before confirming
- **Image previews**: Terminal image rendering with chafa (if installed)

## Keyboard shortcuts

Depending on the screen:

- **Home**: `g` (generate), `m` (models), `a` (account), `h` (history)
- **Generation**: `Enter` (next step), `Esc` (cancel)
- **Gallery**: `o` (open image), `c` (copy path), `d` (delete)

## Installing chafa for image previews

The TUI can render images directly in the terminal if you have [chafa](https://hpjansson.org/chafa/) installed:

<CodeGroup>
```bash macOS
brew install chafa
```

```bash Ubuntu/Debian
sudo apt install chafa
```

```bash Fedora
sudo dnf install chafa
```

```bash Arch
sudo pacman -S chafa
```
</CodeGroup>

## Accessibility

The TUI supports:
- Screen readers via terminal accessibility features
- High contrast mode (respects terminal theme)
- Keyboard-only navigation

## Troubleshooting

<Accordion title="TUI doesn't start">
Ensure your terminal supports ANSI colors and has a minimum width of 80 characters. Try resizing your terminal window.
</Accordion>

<Accordion title="Images don't preview">
Install `chafa` for terminal image rendering. The TUI will fall back to showing file paths if chafa is unavailable.
</Accordion>

<Accordion title="Navigation is sluggish">
The TUI uses React rendering. On slower terminals, try reducing animation or use the CLI for faster workflows.
</Accordion>

<Accordion title="Can't quit the TUI">
Press `q` from the home screen, or use `Ctrl+C` as a fallback.
</Accordion>

## Implementation details

The TUI is built with:
- **Ink**: React for terminal interfaces
- **@inkjs/ui**: UI components (Select, TextInput, Spinner, etc.)
- **React hooks**: State management and API integration
- **Chafa integration**: Image rendering via `renderImageDirect()`

Source files:
- `src/tui.tsx` — TUI entry point
- `src/app.tsx` — Main app component and routing
- `src/screens/` — Individual screen components
- `src/components/` — Reusable UI components
- `src/hooks/` — API and state hooks
